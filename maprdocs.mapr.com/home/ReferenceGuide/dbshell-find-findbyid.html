dbshell find or findbyid To query JSON documents in MapR-DB shell, use either the find or findbyid command. The find command allows you to scan complete tables and retrieve rows that satisfy projection and/or condition clauses. The findbyid command is used to retrieve a single document with a given id. The find examples in this section are sometimes shown split across multiple lines for readability. You must enter the commands on a single line when you run them in dbshell. Note: If your find query requires the [1]OJAI Distributed Query Service, you must install the mapr-drill-internal package on the nodes where you run dbshell. The package is available in the MapR repository where you download MapR Expansion Packs. See [2]MapR Repositories and Packages for details. Syntax find findbyid
--id Parameters find Options findbyid Options Description *, --t, --table *, --t, --table (Required for both find and findbyid) Table path --id --id (Required for findbyid) Document ID. For conditions on a single Document ID, the ID can be provided either by using the --id switch or within a condition payload. For example, --id id1 is equivalent to --c {"$eq":{"_id":"id1"}} Note: Repeating _ids with a switch and within a condition payload is not allowed. --fromid n/a Document ID to start from (inclusive) --toid n/a Document ID to stop at (exclusive) --limit n/a Maximum number of documents to return --withtags, --withTags --withtags, --withTags Enables/disables printing with extended type tags Value: True|False Default: True --pretty --pretty Enables/disables pretty printing of documents Value: True|False Default: True --offset n/a Skips first n number of documents in the result --orderby n/a Sorts the result by the given fields. Specify sort order as either ascending or descending using the keywords, ASC and DESC. Default:ASC Note: The keywords ASC and DESC are case insensitive. Syntax: find
--orderby : See [3]Query with --orderby for examples. --c, --where --c, --where Condition in JSON format. See [4]OJAI Query Condition Syntax for a description of the syntax. See [5]Return Documents Using Projection and Conditions for a dbshell example that uses a condition specified in JSON format. --f, --fields --f, --fields Projections in JSON documents. See [6]JSON Document Field Paths for details about how to specify field paths. See [7]Return Documents Using Projection and Conditions for an example. --q, --query n/a Query JSON documents. This option takes in a query string in JSON format with predefined keywords that define the behavior of the query. For example: find table/test --q { "$select":"a", "$limit":2, "$offset":1 } Note: The find command does not allow --query to work in tandem with other options such as --fields, --where, and --orderby. For example, the following command does not return your desired results: find table/test --f "a" --q {"$limit":2} In addition, repetition of keywords in the --query option is not supported. You should not enter the same keyword twice: // Incorrect {"$select":"a","$select":"b"} // Correct {"$select":["a","b"]} See [8]Query with --query for more examples.
