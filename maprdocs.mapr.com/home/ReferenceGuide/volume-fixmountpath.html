volume fixmountpath Corrects the mount path of a volume. Permissions required: fc or m on the volume. The CLDB maintains information about the mount path of every volume. If a directory in a volume's path is renamed (by a hadoop fs command, for example) the information in the CLDB will be out of date. The volume fixmountpath command does a reverse path walk from the volume and corrects the mount path information in the CLDB. Syntax CLI maprcli volume fixmountpath -name [ -cluster ] REST http[s]://:/rest/volume/fixmountpath? Parameters Parameter Description name The volume name. cluster The cluster name Examples Fix the mount path of volume v1: CLI maprcli volume fixmountpath -name v1 REST https://r1n1.sj.us:8443/rest/volume/fixmountpath?name=v1
