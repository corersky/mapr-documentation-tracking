dbshell insert Description To add documents to JSON tables, use the insert command. Specify the ID of the document in either of two ways: * As the value of the _id field in the document * As the value of the --id parameter of the insert command Parameters CAPTION: Table 1. insert Options Description *, --t, --table (Required) Table path --id Document ID. For rowID-only conditions, the rowID can be provided either within a condition payload or by using the --id switch. For example, --id id1 is equivalent to --c {"$eq":{"_id":"id1"}} Note: Repeating _ids with a switch and within a condition payload is not allowed. -v, --value (Required) JSON document to insert or replace. -c, --where Condition in JSON format. Syntax insert --table --value '{"_id": "}' insert --table
--id --value '{"_id": "}' Example: Insert with _id field The following examples insert a document into a table using a _id field value. insert /data/movies --value '{"_id":"movie0000002", "title":"Developers on the Edge", "studio":"Command Line Studios"}' insert /tables/users --value '{"_id":"user001", "first_name":"John", "last_name":"Doe"}' Example: Insert with --id parameter The following examples insert a document into a table using a value of the --id parameter of the insert command. insert /data/movies --id movie0000003 --value '{"title":"The Golden Master", "studio":"All-Nighter"}' insert /tables/users --id user002 --value '{"first_name":"Jane", "last_name":"Dane"}'
