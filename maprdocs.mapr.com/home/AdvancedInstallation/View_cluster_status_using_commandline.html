Viewing the Cluster Installation Status Using the Command Line Interface Use the Warden to see your status. 1. Log in to a cluster node. 2. Use the following command to list MapR services: $ maprcli service list logpath displayname name state /opt/mapr/logs/mfs.log FileServer fileserver 0 /opt/mapr/hadoop/hadoop-2.7.0/logs ResourceManager resourcemanager 0 /opt/mapr/hadoop/hadoop-2.7.0/logs JobHistoryServer historyserver 0 /opt/mapr/logs/cldb.log CLDB cldb 0 /opt/mapr/logs/nfsserver.log NFS Gateway nfs 0 /opt/mapr/hadoop/hadoop-2.7.0/logs NodeManager nodemanager 0 /opt/mapr/logs/gateway.log GatewayService gateway 0 /opt/mapr/logs/hoststats.log HostStats hoststats 0 /opt/mapr/apiserver/logs/apiserver.log APIServer apiserver 0 $ maprcli license list $ maprcli disk list -host 3. Next, restart Warden on all remaining nodes using the following command: service mapr-warden restart Warden is then responsible for starting the rest of the services configured on each node.
