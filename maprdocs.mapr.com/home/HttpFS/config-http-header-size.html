(Optional) Configure the HTTP Header Size The maxHttpHeaderSize parameter defines the maximum size of the request and response HTTP header, specified in bytes. If it is not specified, this parameter defaults to 8192 (8KB). When Kerberos security is enabled, you may need to increase this value in the server.xml file: /opt/mapr/httpfs/httpfs-1.0/share/hadoop/httpfs/tomcat/conf/server.xml Example If you do not increase this value, you may encounter errors of the following form: HTTP/1.1 400 Bad Request Note: After making this configuration change, restart the HttpFS server.
