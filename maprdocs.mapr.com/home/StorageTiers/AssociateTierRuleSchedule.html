Associating a Tier, Tiering Rule, and/or Schedule with a Volume You can associate a tier, tiering rule, and/or schedule with a new volume or with an already tiering-enabled volume using MCS, CLI, and REST API. Associating a Tier, Tiering Rule, and/or Schedule with a Volume Using the MapR Control System Perform the steps in the following topics to associate a tier, tiering rule, and/or schedule with: * [1]A new volume * [2]A tiering-enabled volume Associating a Tier, Tiering Rule, and/or Schedule with a Volume Using the CLI Run the following command to associate a tier, tiering rule, and/or schedule with: * A new volume: $maprcli volume create -name -path -tieringenable tr ue -tiername -tieringrule -offloadschedule For more information, see [3]volume create. * An already tiering-enabled volume: $maprcli volume modify -name -tiername -tieringrule -offloadschedule For more information, see [4]volume modify. Note: You cannot modify a tier associated with a volume after it is set. Associating a Tier, Tiering Rule, and/or Schedule with a Volume Using the REST API Send a request of type POST to associate a tier, tiering rule, and/or schedule with: * A new volume. For example: curl -k -X POST 'https://:8443/rest/volume/create?name=&path=&tieringenable=true&tiername=&tieringrule=&of floadschedule=' --user mapr:mapr For more information, see [5]volume create. * An already tiering-enabled volume. For example: curl -k -X POST 'https://:8443/rest/volume/modify?name=&tiername =&tieringrule=&offloadschedule=' --user mapr: mapr For more information, see [6]volume modify. Note: You cannot modify a tier associated with a volume after it is set.
