Enabling Impersonation on a Non-Secure Cluster To enable impersonation on a non-secure cluster, follow these steps: 1. Install the mapr-hbase package on a cluster that is running version 4.0.2 or later. This package contains all of the HBase binaries. For installation details, see [1]HBase Client and MapR-DB. 2. Enable simple authentication via the REST Gateway by appending the following property to the hbase-site.xml file (/opt/mapr/hbase/hbase/conf/hbase-site.xml): The simple authentication protocol is a Hadoop pseudo authenticator that serves as an example and is part of the hadoop-common package. hbase.rest.authentication.type simple 3. Set the following environment variable to enable impersonation: export MAPR_IMPERSONATION_ENABLED=1 4. Start the REST Gateway server as the MAPR_USER. /opt/mapr/hbase/hbase/bin/hbase-daemon.sh start rest -p port References 1. file://localhost/root/docsync/tmp/maprdocs.mapr.com/home/HBase/HBase.html
