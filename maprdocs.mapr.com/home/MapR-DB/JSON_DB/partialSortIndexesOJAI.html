Avoiding Partial Sorts with Secondary Indexes in OJAI Partial sorts can occur due to secondary index lags. This section describes how to avoid this in your OJAI application. Set the following option to TRUE using the following OJAI method: * [1]Java query.setOption("ojai.mapr.query.force-noncovering-sort", true); This will force the Query Service to explicitly sort the data. Do not set this option if you do not expect to encounter index lags. Otherwise, you lose the ordering advantage that secondary indexes provide. For more information about why partial sorts occur, see [2]Partial Sorts with Non-Covering Indexes.
