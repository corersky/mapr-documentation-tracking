Query Conditions in OJAI Applications You can create a query condition in an OJAI application in either of two ways. One way is to create an OJAI QueryCondition object and call methods in the class to construct your query condition. Another way is to create an OJAI query condition in a JSON format. Example: Creating a QueryCondition Object The following example shows how to define a QueryCondition object for this query condition: (a.b.[0].boolean == false && (a.c.d != 5 || a.b[1].decimal > 1 || a.b[1].decimal < 10)) * [1]Java QueryCondition qc = connection.newCondition() .and() .is("a.b[0].boolean", Op.EQUAL, false) .or() .is("a.c.d", Op.NOT_EQUAL, 5) .is("a.b[1].decimal", Op.GREATER, 1) .is("a.b[1].decimal", Op.LESS, 10) .close() .close() .build(); Pass the QueryCondition object to the [2]Query.where method. See the Java - OJAI QueryCondition Object example at [3]Querying with Conditions for a complete Java code example. Creating a Query Condition in JSON Format To create a query condition in a JSON format, use the [4]OJAI Query Condition Syntax. The following syntax shows how to create this query condition: (a.b.[0].boolean == false && (a.c.d != 5 || a.b[1].decimal > 1 || a.b[1].decimal < 10)) * [5]Java This is a Java string for the condition: String jc = new String( '{ \ "$and":[ \ {"$eq":{"a.b[0].boolean":false}}, \ {"$or":[ \ {"$ne":{"a.c.d":5}}, \ {"$gt":{"a.b[1].decimal":1}}, \ {"$lt":{"a.b[1].decimal":10}} \ ]} \ ]} \ )'; Pass the string to the [6]Query.where method. See the Java - OJAI Query Condition in JSON Format example at [7]Querying with Conditions for a complete Java code example.
