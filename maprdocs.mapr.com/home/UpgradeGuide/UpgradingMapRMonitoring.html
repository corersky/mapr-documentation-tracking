Upgrading MapR Monitoring Complete the following steps to upgrade MapR Monitoring without the MapR Installer. Execute the following commands as root or using sudo. 1. Upgrade the following metric monitoring packages wherever they are installed on the cluster: mapr-collectd, mapr-grafana, and mapr-opentsdb. For example, on a three node cluster, you could run the following commands to upgrade metrics monitoring packages: + For CentOS/RedHat: o Node A: yum upgrade mapr-collectd mapr-grafana o Node B: yum upgrade mapr-collectd mapr-opentsdb o Node C: yum upgrade mapr-collectd + For Ubuntu: o Node A: apt-get install mapr-collectd mapr-grafana o Node B: apt-get install mapr-collectd mapr-opentsdb o Node C: apt-get install mapr-collectd + For SUSE: o Node A: zypper update mapr-collectd mapr-grafana o Node B: zypper update mapr-collectd mapr-opentsdb o Node C: zypper update mapr-collectd 2. Upgrade the following log monitoring packages wherever they are installed on the cluster: mapr-fluentd, mapr-elasticsearch, and mapr-kibana. For example, on a three node MapR cluster, you can run the following commands to upgrade log monitoring packages: + For CentOS/RedHat: o Node A: yum upgrade mapr-fluentd mapr-elasticsearch o Node B: yum upgrade mapr-fluentd mapr-elasticsearch o Node C: yum upgrade mapr-fluentd mapr-elasticsearch mapr-kibana + For Ubuntu: o Node A: apt-get install mapr-fluentd mapr-elasticsearch o Node B: apt-get install mapr-fluentd mapr-elasticsearch o Node C: apt-get install mapr-fluentd mapr-elasticsearch mapr-kibana + For SUSE: o Node A: zypper update mapr-fluentd mapr-elasticsearch o Node B: zypper update mapr-fluentd mapr-elasticsearch o Node C: zypper update mapr-fluentd mapr-elasticsearch mapr-kibana
