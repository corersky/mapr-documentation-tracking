Integrate Hue with Oozie Complete the following steps to integrate Hue with Oozie: 1. Edit the hue.ini file to configure the location where the Oozie service is running. The path to the hue.ini file is: /opt/mapr/hue/hue-/desktop/conf/hue.ini. [liboozie] # The URL (host IP address) where the Oozie service is running. This is required in order for # users to submit jobs. oozie_url=http://:/oozie The [1] depends on whether your cluster is secure. 2. Enable user impersonation for Oozie through Hue: a. Add the following lines to the oozie-site.xml file (the path to the file is: /opt/mapr/oozie/oozie-4.2.0/conf/oozie-site.xml): oozie.service.ProxyUserService.proxyuser..hosts * oozie.service.ProxyUserService.proxyuser..groups * In most cases, mapr is the . This should be the same default user that is specified in the hue.ini and the core-site.xml. b. Restart Oozie. To restart Oozie, first stop Oozie then start it: maprcli node services -name oozie -action restart -nodes To verify that the Oozie server started, enter: lsof -i: The output from this command should look similar to this: COMMAND PID USER FD TYPE DEVICE SIZE/OFF NODE NAME java 16644 mapr 35u IPv6 6992 6776 0t0 TCP *:irisa (LISTEN) You can also check Oozie logs to verify that Oozie started. The log is found here: /opt/mapr/oozie/oozie-/logs/oozie.log 3. Perform any additional Hue configurations and then restart Hue so that the changes will take effect. See [2]Starting the Hue Webserver.
