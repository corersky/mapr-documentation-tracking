Configure a MySQL Data Store for Oozie Follow these steps to configure Oozie to use a MySQL database as the Oozie data store (instead of the default Apache Derby database). The MySQL database can also be used to support [1]high availability (HA). 1. Using the mysql command-line tool, create a MySQL database, user, and password for the Oozie user. For example: mysql> create database oozie; mysql> grant all privileges on oozie.* to ''@'%' identified by ''; 2. Stop the Oozie server: maprcli node services -name oozie -action stop -nodes 3. Set the following JPAService properties in the oozie-site.xml file: Note: In the JDBC URL property, use the correct hostname (where MySQL is running). oozie.service.JPAService.jdbc.driver com.mysql.jdbc.Driver oozie.service.JPAService.jdbc.url jdbc:mysql://localhost:3306/oozie oozie.service.JPAService.jdbc.username oozie oozie.service.JPAService.jdbc.password oozie 4. Copy the MySQL JDBC driver (mysql-connector-java--bin.jar) to the following directory. /opt/mapr/oozie/oozie-/libext Starting from MEP-4.1.0, this step is not needed as the mysql-connector-java--bin.jar file is automatically copied from the /opt/mapr/lib/ directory to the /opt/mapr/oozie/oozie-/libext directory. 5. Starting from MEP-4.1.0, this step is not needed. Execute the oozie-setup.sh script: /opt/mapr/oozie/oozie-/bin/oozie-setup.sh -hadoop /opt/mapr/h adoop/hadoop- Parameter Description oozie- Specify the version from the directory path. -hadoop Specify the hadoop version. hadoop- Specify the version from the directory path. 6. Start the Oozie server: maprcli node services -name oozie -action start -nodes 7. Check that MySQL is now in use by looking at the contents of the oozie.log file. For example: cat /opt/mapr/oozie/oozie-/logs/oozie.log |grep mysql 2015-07-24 06:10:07,023 INFO JPAService:541 - SERVER[local.novalocal] USER[-] GROUP[-] TOKEN[-] APP[-] JOB[-] ACTION[-] JPA configuration: DriverClassName=com.mysql.jdbc.Driver,Url=jdbc:mysql://localhost:3306/oozie,User name=oozie,... 8.
