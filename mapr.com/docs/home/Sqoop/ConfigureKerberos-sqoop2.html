Configure Kerberos Authentication for Sqoop2 You can configure Sqoop2 to use Kerberos authentication. When Sqoop2 uses Kerberos authentication, the cluster and other components that work with Sqoop2, such as Hue, must also use Kerberos authentication. Note the following items when you complete the configuration steps: * Replace with the FQDN of the server. To determine this value, run hostname -f in the command line. * Replace with the realm name in the krb5.conf file, which is generated when you install the KDC server on the cluster. Configuring Kerberos Authentication for Sqoop2: 1. Using the kadmin program, run the following commands to create principals for Sqoop 2: addprinc -randkey HTTP/@ addprinc -randkey mapr/@ Kerberos uses the principal HTTP/@ for communication between Sqoop2 client and Sqoop2 server. The principal mapr/@ is the Sqoop2 user that communicates between Sqoop2 server and MapR Filesystem. 2. Using the kadmin program, run the following commands to create keytabs for the principals: xst -k /opt/mapr/conf/mapr.keytab HTTP/@ xst -k /opt/mapr/conf/mapr.keytab mapr/@ 3. Modify the following properties in Sqoop2 configuration file (sqoop.properties). In Sqoop 1.99.6, the sqoop.properties file is in the following directory: /opt/mapr/sqoop/sqoop-/server/conf/. In Sqoop 1.99.7, the sqoop.properties file is in the following directory: /opt/mapr/sqoop/sqoop-/conf/. org.apache.sqoop.security.authentication.type=KERBEROS org.apache.sqoop.security.authentication.handler=org.apache.sqoop.security.authe ntication.KerberosAuthenticationHandler org.apache.sqoop.security.authentication.kerberos.principal=mapr/@ org.apache.sqoop.security.authentication.kerberos.keytab=/opt/mapr/conf/mapr.key tab org.apache.sqoop.security.authentication.kerberos.http.principal=HTTP/@ org.apache.sqoop.security.authentication.kerberos.http.keytab=/opt/mapr/conf/map r.keytab org.apache.sqoop.security.authentication.enable.doAs=true org.apache.sqoop.security.authentication.proxyuser.mapr.users=* 4. Start Sqoop2 server. maprcli node services -name sqoop2 -action start -nodes 5. Using the kinit program, run the following command to generate a ticket: kinit HTTP/@ -kt /opt/mapr/conf/mapr.keytab 6. Start the Sqoop2 client. sudo -u mapr /opt/mapr/sqoop/sqoop-/bin/sqoop.sh client
