MapR Alarm Metrics Every 10 seconds, the collectd service uses a MapR plugin to gather the cluster alarms. CAPTION: Name Description Additional Tag(s) mapr.alarms.alarm_raised The number of alarms raised. The timestamp for each alarm is based on the time that MapR raised the alarm, not the time when the collectd service gathered the alarm data. * alarm_name: Display values for a specified alarm name. * alarm_entity:. Display values for a specified volume, node, user, or group.
