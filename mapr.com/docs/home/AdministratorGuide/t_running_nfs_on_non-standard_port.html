Running NFS on a Non-standard Port 1. To run NFS on an arbitrary port, modify the following line in warden.conf: service.command.nfs.start=/etc/init.d/mapr-nfsserver start Add -p to the end of the line, as in the following example: service.command.nfs.start=/etc/init.d/mapr-nfsserver start -p 12345 2. After modifying warden.conf, restart the MapR NFS server by issuing the following command: maprcli node services -nodes -nfs restart 3. You can verify the port change with the rpcinfo -p localhost command.
