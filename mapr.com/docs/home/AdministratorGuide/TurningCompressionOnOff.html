Turning Compression On or Off on Directories Using the CLI You can turn compression on or off for a given directory in two ways: * Set the value of the Compression attribute in the [1].dfs_attributes file at the top level of the directory. + Set Compression=lzf|lz4|zlib to turn compression on for a directory. + Set Compression=false to turn compression off for a directory. * Use the command [2]hadoop mfs -setcompression on|off/lzf/lz4/zlib . If you choose -setcompression on without specifying an algorithm, lz4 is used by default. This algorithm has improved compression speeds for MapR's block size of 64 KB. The symbols for the various compression settings are explained here: CAPTION: Symbol Compression Setting Z lz4 z zlib L lzf U Uncompressed, or previously compressed by another algorithm Example Suppose the volume test is NFS-mounted at /mapr/my.cluster.com/projects/test. You can turn off compression by editing the file /mapr/my.cluster.com/projects/test/.dfs_attributes and setting Compression=false. To accomplish the same thing from the hadoop shell, use the following command: hadoop mfs -setcompression off /projects/test You can view the compression settings for directories using the [3]hadoop mfs -ls command.
