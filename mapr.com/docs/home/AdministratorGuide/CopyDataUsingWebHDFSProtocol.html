Copying Data Using the webhdfs:// Protocol Before you can copy data from an HDFS cluster to a MapR cluster using the webhdfs:// protocol, you must configure the MapR cluster to access the HDFS cluster. To do this, complete the steps listed in [1]Configuring a MapR Cluster to Access an HDFS Cluster for the security scenario that best describes your HDFS and MapR clusters and then complete the steps listed under [2]Verifying Access to an HDFS Cluster. The HDFS cluster must have WebHDFS enabled. Verify that the following parameter exists in the hdfs-site.xml file and that the value is set to "true." dfs.webhdfs.enabled true You also need the following information: * - the IP address or hostname of the NameNode in the HDFS cluster * - the HTTP port on the NameNode in the HDFS cluster * - the path to the HDFS directory from which you plan to copy data * - the path in the MapR cluster to which you plan to copy HDFS data To copy data from HDFS to MapR-FS using the webhdfs:// protocol, complete the following step: Run the following command from a node in the MapR cluster: hadoop distcp webhdfs://:/ maprfs:/// Example hadoop distcp webhdfs://nn2:50070/user/sara maprfs:///user/sara Note: The triple slashes in 'maprfs:///...' are required.
