Setting Default Volume Topology Using the CLI By default, new volumes are created with a topology of /data. To change the default topology, use the [1]config save command to change the cldb.default.volume.topology configuration parameter. Example: maprcli config save -values "{\"cldb.default.volume.topology\":\"/data/rack02\"} " After running the above command, new volumes have the volume topology /data/rack02 by default, which could be useful to restrict new volume data to subset of the cluster.
