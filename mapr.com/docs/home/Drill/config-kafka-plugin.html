Configuring the Kafka Storage Plugin The Kafka storage plugin is not officially supported for Drill on MapR Streams. However, if you choose to configure Kafka as a data source in Drill, you must update the /jars/3rdParty directory so it contains the required JAR files and then restart Drill before you configure the kafka storage plugin in the Drill Web Console. Verify that the nodes in your cluster meet the requirements and then complete the steps listed. Requirements The Kafka storage plugin requires: * A MapR 6.0.1 cluster. * Drill 1.13 installed on nodes. * The MapR Kafka client package (mapr-kafka-1.0.1) installed on at least one node. The Kafka client installation provides the kafka JAR files that you copy into the /jars/3rdParty directory (step 4). Steps Complete the following steps to make Kafka a consumable resource for Drill on MapR Streams: Note: Do not perform step 2 if you installed Drill using the MapR RPM or Debian packages. Drill contains symbolic links to the files in /opt/mapr/lib. Step 2 is only required if you installed Drill using a TAR file. 1. Remove the specified JAR files from the /jars/3rdParty directory based on the Drill installation method: + If you installed Drill using MapR RPM or Debian packages, only remove JAR files that start with kafka, such as kafka_-mapr.jar, from the /jars/3rdParty directory. Drill contains symbolic links to the required mapr JAR files in /opt/mapr/lib. + If you installed Drill using a TAR file, remove all the JAR files that start with mapr and kafka, such as maprdb--mapr.jar, maprfs--mapr.jar, and kafka_-mapr.jar, from the /jars/3rdParty directory. 2. (Only perform this step if you installed Drill using a TAR file.) Copy the following mapr JAR files from the /opt/mapr/lib directory into /jars/3rdParty directory: + maprdb-6.0.1-mapr.jar + maprdb-6.0.1-mapr-tests.jar + maprfs-6.0.1-mapr.jar + maprfs-6.0.1-mapr-tests.jar + mapr-hbase-6.0.1-mapr.jar + mapr-hbase-6.0.1-mapr-tests.jar 3. Copy the mapr-streams-6.0.1-mapr.jar file from the /opt/mapr/lib directory into the /jars/3rdParty directory. 4. Copy the following kafka JAR files from the /opt/mapr/kafka/kafka-1.0.1/libs directory into the /jars/3rdParty directory: + kafka_2.11-1.0.1-mapr-1803.jar + kafka-clients-1.0.1-mapr-1803.jar 5. Issue the following command to restart Drill: $ maprcli node services -name drill-bits -action restart -nodes 6. Log in to the[1]Drill Web Console, and configure the kafka storage plugin. See[2]Kafka Storage Plugin for instructions.
