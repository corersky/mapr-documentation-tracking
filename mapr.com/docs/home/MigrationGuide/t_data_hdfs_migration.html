Using the hdfs:// Protocol This section describes how to copy data from an HDFS cluster to a MapR cluster using the hdfs:// protocol. Before you can copy data from an HDFS cluster to a MapR cluster using the hdfs:// protocol, you must configure the MapR cluster to access the HDFS cluster. To do this, complete the steps listed in [1]Configuring a MapR Cluster to Access an HDFS Cluster for the security scenario that best describes your HDFS and MapR clusters and then complete the steps listed under [2]Verifying Access to an HDFS Cluster. You also need the following information: * : the IP address or hostname of the NameNode in the HDFS cluster * : the port for connecting to the NameNode in the HDFS cluster * : the path to the HDFS directory from which you plan to copy data * : the path in the MapR cluster to which you plan to copy HDFS data * : a file in the HDFS path To copy data from HDFS to MapR-FS using the hdfs:// protocol, complete the following steps: 1.
