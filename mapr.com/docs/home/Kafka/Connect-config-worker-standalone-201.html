Kafka Connect 2.0.1: Standalone Worker Configuration Options This section describes worker parameters that are specific to standalone configurations. The offset.storage.file.filename and rest.port parameter are specific to the standalone worker configuration. These parameters are sent in the connect-standalone.properties file. CAPTION: Table 1. Standalone Configuration Parameters Parameter Description rest.port Port the REST interface listens on for HTTP requests. If you run multiple standalone instances on the same host, this setting must have different values for each instance. Type: int. Default: 8083 offset.storage.file.filename The file to store connector offsets in. By storing offsets on disk, a standalone process can be stopped and started on a single node and resume where it previously left off. Type: string. Default: empty
