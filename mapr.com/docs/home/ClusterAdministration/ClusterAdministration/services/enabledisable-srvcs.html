Enabling and Disabling a Service Using the CLI and REST API You can disable a service to prevent it from starting or restarting when Warden starts or restarts and enable a service to allow it to start or restart when Warden starts or restarts. Disabling a Service Using the CLI and REST API * [1]CLI * [2]REST Run the following command: maprcli node services -nodes -name -action disable Send a request of type POST. For example: curl -k -X POST 'https://:8443/rest/node/services?nodes=&name=&action=disable' --user mapr:mapr Note: When you disable a service, the service is stopped and the service will not be automatically starrted/restarted when Warden is started/restarted. See [3]node services for more information. Enabling a Service Using the CLI and REST API * [4]CLI * [5]REST Run the following command: maprcli node services -nodes -name -action enable Send a request of type POST. For example: curl -k -X POST 'https://:8443/rest/node/services?nodes=&name=&action=enable' --user mapr:mapr Note: When you enable a service, the service is started/restarted when Warden is started/restarted. See [6]node services for more information.
