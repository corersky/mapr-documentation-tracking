Starting, Stopping, and Restarting MapR NFSv3 You can start, stop, and restart NFS using MCS and the CLI. Starting, Stopping, and Restarting MapR NFSv3 Using the MapR Control System See: * [1]Starting the Services on the Cluster Using the MapR Control System * [2]Stopping a Service on the Cluster Using the MapR Control System * [3]Restarting the Services on the Cluster Using the MapR Control System Starting, Stopping, and Restarting MapR NFSv3 Using the CLI and REST API * [4]NFSv3 Server * [5]REST The command to stop, start, or restart MapR NFSv3 server is: maprcli node services -nodes -nfs stop|start|restart Send a request of type POST. For example: curl -k -X POST 'https://:8443/rest/node/services?nodes=&nfs=st op|start|restart' -- user mapr:mapr Note: When NFS server is stopped, the VIPs associated with the server are released and CLDB attempts to reassign the VIPs to other available NFS servers. For the complete list of parameters, see [6]node services.
