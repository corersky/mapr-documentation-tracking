MapR Monitoring Components - MEP 4.1.0 Release Notes The notes below relate specifically to the MapR Converged Data Platform. Version The MEP 4.1.0 release contains the following monitoring component versions: * Collectd 5.7.2 * Fluentd 0.14.20 * Opentsdb 2.4.0 * Elasticsearch 5.4.1 * Grafana 4.4.2 * Kibana 5.4.1 Release Date February 2018 MapR Version Interoperability [1]MEP Components and OS Support Package Names [2]Package Names for MapR Expansion Packs (MEPs) New in This Release MapR Monitoring in the MEP 4.1.0 release introduces the following enhancements: * File permissions changed to ensure compliance with the MapR Security Policy. * The following rules are added for specifying metric names: + Strings are case sensitive: for example, "Sys.Cpu.User" is stored separately from "sys.cpu.user". + Spaces are not allowed. + Only the following characters are allowed: a to z, A to Z, 0 to 9, -, _, ., / or Unicode letters (as per the specification). + Metric and tags are not limited in length, though you must try to keep the values fairly short. Patches None Known Issues and Limitations See [3]Known Issues at Release (MapR 6.0.0) Resolved Issues * FLUD-17: Fluent SSL verification enabled for HTTPS requests. * FLUD-21: Administrator passwords removed from the FluentD log files. * FLUD-20: Log rotate is fixed.
