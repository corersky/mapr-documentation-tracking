Setting Query Options in Node.js OJAI This topic lists and describes the query options you can set in your Node.js OJAI application. It also describes how to set query options in the MapR Data Access Gateway. Option Name Description ojai.mapr.query.include-query-plan Enables or disables availability of the query plan for retrieval Value: true|false Default: false ojai.mapr.query.timeout-milliseconds Query timeout in milliseconds Maximum allowed value is 2147483647. Default: None; no timeout To set any of these query options, you can pass the option as the second parameter in the DocumentStore.find method. The following code snippet sets the query timeout to 3000 milliseconds: const docStream = store.find( query, { 'ojai.mapr.query.timeout-milliseconds': 3000 } ); Setting Query Options Through the MapR Data Access Gateway There are some options (for example, the max client sort limit) that apply to all MapR Data Access Gateway clients, including Node.js OJAI clients. To modify these options, you must update the file /opt/mapr/data-access-gateway/conf/ojai-config.json. See [1]Application Properties for details.
