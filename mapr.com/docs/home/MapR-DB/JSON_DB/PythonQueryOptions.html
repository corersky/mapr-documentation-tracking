Setting Query Options in Python OJAI This topic lists and describes the query options you can set in your Python OJAI application. It also describes how to set query options in the MapR Data Access Gateway. Option Name Description ojai.mapr.query.include-query-plan Enables or disables availability of the query plan for retrieval Value: True|False Default: False ojai.mapr.query.result-as-document Enables or disables returning the query result as an OJAI Document class object versus a Python dictionary Value: True|False Default: False; returns query result as a Python dictionary ojai.mapr.query.timeout-milliseconds Query timeout in milliseconds Maximum allowed value is 2147483647. Default: None; no timeout To set any of these query options, you can pass the option as the second parameter in the [1]DocumentStore.find method. The following code snippet sets the option to return the query result as a [2]Document object: options = {'ojai.mapr.query.result-as-document': True} query_result = store.find(query, options=options) Setting Query Options Through the MapR Data Access Gateway There are some options (for example, the max client sort limit) that apply to all MapR Data Access Gateway clients, including Python OJAI clients. To modify these options, you must update the file /opt/mapr/data-access-gateway/conf/ojai-config.json. See [3]Application Properties for details.
