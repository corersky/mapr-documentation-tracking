Forcing Secondary Index Usage in Java OJAI To force the MapR client to use an index, set the ojai.mapr.query.hint-using-index query option, passing in the name of the index as the second parameter: query.setOption("ojai.mapr.query.hint-using-index", indexName); Regardless of cost considerations, the MapR client attempts to use the specified index. To use the index, the index must benefit filter conditions, the order by, or projections in the query as described at [6]Queries that Benefit from Secondary Indexes. Otherwise, the MapR client ignores the option. To force the MapR client to not use any indexes, specify the table name without the full path as the second parameter in the calls shown earlier. For example, if the full path of your table is /mapr/sanfrancisco/volume1/customer, pass the name customer as the second parameter. Note: Setting this option in your Java OJAI application has no effect if you are using either the OJAI Distributed Query Service or Drill. Avoiding Partial Sorts with Secondary Indexes in Java OJAI Partial sorts can occur due to secondary index lags. To avoid these lags, set the ojai.mapr.query.force-noncovering-sort option to TRUE using the following OJAI method: query.setOption("ojai.mapr.query.force-noncovering-sort", true); This will force the Query Service to explicitly sort the data. Do not set this option if you do not expect to encounter index lags. Otherwise, you lose the ordering advantage that secondary indexes provide. For more information about why partial sorts occur, see [7]Partial Sorts with Non-Covering Indexes. Setting Java OJAI Distributed Query Service Properties OJAI queries may leverage the OJAI Distributed Query Service. To modify OJAI Distributed Query Service property settings in your Java OJAI application, prefix the OJAI Distributed Query Service property name with ojai.mapr.drill. For example, the following call disables using secondary indexes when queries use the Query Service. query.setOption("ojai.mapr.drill.planner.enable_index_planning", false); See [8]Index Planning and Execution Configuration Options for the list of available OJAI Distributed Query Service properties.
