Enabling Impersonation on a Secure Cluster This section describes tasks to enable impersonation on a secure cluster. To enable impersonation on a secure cluster, follow these steps: 1. Install the mapr-hbase package on a cluster that is running version 4.0.2 or later. This package contains all of the HBase binaries. For installation details, see [1]HBase Client and MapR-DB. 2. Enable simple authentication via the REST Gateway by appending the following property to the hbase-site.xml file (/opt/mapr/hbase/hbase/conf/hbase-site.xml): The simple authentication protocol is a Hadoop pseudo authenticator that serves as an example and is part of the hadoop-common package. hbase.rest.authentication.type simple 3. Set the following variable in /opt/mapr/conf/env.sh: export MAPR_HBASE_SERVER_OPTS="${HYBRID_LOGIN_OPTS} ${MAPR_SSL_OPTS} -Dmapr.library.flatclass 4. Start the REST Gateway server as the MAPR_USER. /opt/mapr/hbase/hbase/bin/hbase-daemon.sh start rest -p port References 1. file://localhost/root/docsync/tmp/mapr.com/docs/home/HBase/HBase.html
