Steps for Configuring the MapR Data Fabric for Kubernetes FlexVolume Driver This page summarizes the high-level steps for configuring the MapR Data Fabric for Kubernetes FlexVolume Driver to provide static or dynamic provisioning. To learn more about static and dynamic provisioning, see [1]Static and Dynamic Provisioning Using FlexVolume Driver. Static Provisioning 1. Install the MapR Data Fabric for Kubernetes. See [2]Installing the MapR Data Fabric for Kubernetes FlexVolume Driver. 2. In your Pod spec or as part of a separate configuration file, configure a PersistentVolume. See [3]Example: Mounting a PersistentVolume for Static Provisioning Using the FlexVolume Driver and [4]Persistent Volumes. 3. Do one of the following: + Annotate the Pod spec to provide information about the MapR volume. See [5]Example: Statically Provisioning a MapR Volume Using the FlexVolume Plug-in. + In your Pod spec or as part of a separate configuration file, configure a PersistentVolumeClaim. See [6]PersistentVolumeClaims. 4. Run the Pod spec by using kubectl commands. See [7]Overview of kubectl. Dynamic Provisioning 1. Install the MapR Data Fabric for Kubernetes. See [8]Installing the MapR Data Fabric for Kubernetes FlexVolume Driver. 2. In your Pod spec or in a separate configuration file, create a storage class. See [9]Example: Mounting a PersistentVolume for Dynamic Provisioning Using the FlexVolume Driver and [10]Storage Classes. 3. In your Pod spec or in a separate configuration file, configure a PersistentVolumeClaim. See [11]Example: Mounting a PersistentVolume for Dynamic Provisioning Using the FlexVolume Driver. 4. Run the Pod spec by using kubectl commands. See [12]Overview of kubectl.
