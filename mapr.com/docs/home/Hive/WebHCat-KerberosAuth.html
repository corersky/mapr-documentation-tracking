Configure Kerberos Authentication for WebHCat When security features are enabled on your MapR cluster and Kerberos is in use, communications between WebHCat and its clients can use Kerberos with [1]SPNEGO. To enable WebHCat to use Kerberos, complete the following steps on the node where WebHCat is installed. 1. Create the principal HTTP/ for WebHCat and add the principal to the keytab file. For example: kadmin: addprinc -randkey HTTP/ kadmin: xst -k /opt/mapr/HTTP.keytab HTTP/ 2. Verify the following: + The principal was added to the /opt/mapr/conf/HTTP.keytab file and that the file is only readable by the mapr user. For example: chown mapr /opt/mapr/conf/HTTP.keytab + The node where the WebHCat server is running has an HTTP user with a valid maprlogin password. 3. Add the following section to the /opt/mapr/hive/hive-/hcatalog/etc/webhcat/webhcat-site.xml file: templeton.kerberos.secret secret value templeton.kerberos.principal HTTP/ templeton.kerberos.keytab /opt/mapr/conf/HTTP.keytab 4. Add the following section to the /opt/mapr/hadoop/hadoop-/etc/hadoop/core-site.xml file: hadoop.proxyuser.HTTP.groups * Allow the superuser mapr to impersonate any member of any gro up hadoop.proxyuser.HTTP.hosts * The superuser can connect from any host to impersonate a user 5. Start WebHCat. See Managing the WebHCat Server. 6. To test if the connection is working, generate a Kerberos ticket with the kinit utility and then run the following command: curl --negotiate -i -u : 'http://:50111/templeton/v1/ddl/database/' References 1. file://localhost/root/docsync/tmp/mapr.com/docs/home/SecurityGuide/ConfiguringSPNEGOonMapR.html#ConfiguringSPNEGOonMapR
