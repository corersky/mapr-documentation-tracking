Specifying a Storage Policy You can associate a storage policy with a tiering-enabled volume using the MapR Control System and the CLI. Specifying a Storage Policy Using the MapR Control System You can associate a storage policy with a tiering-enabled volume when you are: * Creating a volume by clicking Create Volume button in the Data > Volumes page. * Editing the tiering-enabled volume by clicking Edit Volume button in the [1]volume information page. To associate a storage policy with the volume, in the Create Volume or Edit Volume page: 1. Click the Browse link associated with the Storage Policy field to display the Browse Storage Policies window. 2. Review the name and detail of each storage policy and choose a storage policy from the list. 3. Click Select to associate the storage policy with the volume. 4. Complete the steps for [2]creating or [3]editing the volume. Specifying a Storage Policy Using the CLI and REST API You can associate a rule with a tiering-enabled volume by specifying the tieringrule parameter with the [4]volume create or [5]volume modify command. * [6]CLI * [7]REST Run a command similar to the following to associate a rule when: * Creating a volume: maprcli volume create -name -path -tieringenable true -tie rname -tieringrule -json For the list of all other required and optional parameters, see [8]volume create. * Editing the volume: maprcli volume modify -name -tieringrule -json For the list of all other required and optional parameters, see [9]volume modify. Send a request of type POST. For example, to associate a rule when: * Creating a volume: curl -k -X POST 'https://:8443/rest/volume/create?name=&path=&tieringenable=true&tiername=&tieringrule=' --user ma pr:mapr For the list of all other required and optional parameters, see [10]volume create. * Editing the volume: curl -k -X POST 'https://:8443/rest/volume/modify?name=&tieringru le=' --user mapr:mapr For the list of all other required and optional parameters, see [11]volume modify.
