Authenticating Azure Data Lake Store Account To access data stored in Azure Data Lake Store (ADLS), you must first authenticate your ADLS account using your ADLS credentials. 1. Obtain the following properties from your Azure application: + dfs.adls.oauth2.access.token.provider.type - Authentication type You can use the ClientCredential method as shown in the example below. You can also use Refresh Tokens or Client Keys. For more information, refer to the [1]open source documentation. + dfs.adls.oauth2.client.id - Application ID that uses ADLS Create an Azure Active Directory application and get your application ID and authentication key. For more information, see [2]Azure documentation. + dfs.adls.oauth2.refresh.url - Token endpoint To get the token endpoint, navigate to Azure Active Directory and click on Endpoints. Use the OAUTH 2.0 TOKEN ENDPOINT value. + dfs.adls.oauth2.credential - Access token Obtain the access token key value from App Registrations in your Azure account. 2. Add the obtained properties to the core-site.xml file: Note: The core-site.xml file can be overwritten using the command line. dfs.adls.oauth2.access.token.provider.type ClientCredential dfs.adls.oauth2.client.id f377fab9-c0a3-4531-a1c9-77345105 dfs.adls.oauth2.refresh.url https://login.microsoftonline.com/25735fb/oauth2/token dfs.adls.oauth2.credential WTkn4xS0ISsqyzo4R6bu/OW2oPyGNMzWRw/d2z2CGiw= 3. Give access to the files for your application. See the [3]Azure documentation on authorization and access control. 4. For secure clusters, MapR-SASL and Kerberos, import only the CA certificate you need. Refer to [4]Configuring MapR for HTTPS Upload to S3 for an example on how to properly consider certificate security.
