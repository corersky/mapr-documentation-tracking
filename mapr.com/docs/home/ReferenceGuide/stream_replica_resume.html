stream replica resume Resumes replication from one stream to another stream Replication can be paused during autosetup and replication phases. When replication resumes, it continues from where it left off. Permissions Required on the Source Cluster To run this command, your user ID must have the following permissions: * [1]readAce and [2]writeAce on the volume * [3]lookupdir on directories in the path * adminperm permission on the source stream Note: The mapr user is not treated as a superuser. MapR Event Store For Apache Kafka does not allow the mapr user to run this command unless that user is given the relevant permission or permissions with access-control expressions. Syntax CLI maprcli stream replica resume -path -replica REST http[s]://:/rest/stream/replica/resume?path=&replica= Parameters Parameter Description path The path and name of the stream that is the source for the replica that you want to resume replicating to. replica The path and name of the stream replica that you want to resume replicating to.
