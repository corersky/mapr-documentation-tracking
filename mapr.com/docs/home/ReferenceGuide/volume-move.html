volume move Moves the specified volume or mirror to a different topology. Permissions required: m or fc on the volume. Syntax CLI volume move [ -cluster cluster_name ] -name volumeName> [ -topology topology ] [ -ectopology ] REST Request Type POST Request URL http[s]://:/rest/volume/move? Parameters Parameter Description cluster The name of the cluster on which to run the command. ectopology The new rack path for the erasure-coded volume if you are moving an erasure-coded volume. This is only required for moving an erasure-coded volume. name The name of the volume to move. For moving: * An erasure coded volume, specify the name of the front-end volume. * The metadata volume associated with a tier, specify the name of the metadata volume. topology The new rack path for the: * Regaular or tiered standard volume if you are moving a regular or tiered standard volume. * Regaular or tiered mirror volume if you are moving a regular or tiered mirror volume. * Metadata volume if you are moving a metadata volume associated with a tier. If you are moving an erasure-coded volume, this is not required.
