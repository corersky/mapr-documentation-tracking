stream purge Runs the purge process, removing messages that are marked for deletion and reclaiming disk space. For information about the purge process, see [1]Time-to-Live for Messages. Note: The mapr user is not treated as a superuser. MapR Event Store For Apache Kafka does not allow the mapr user to run this command unless that user is given the relevant permission or permissions with access-control expressions. Permissions Required To run this command, your user ID must have the following permissions: * [2]readAce and [3]writeAce on the volume * [4]lookupdir on directories in the path * adminperm permission on the stream Note: The mapr user is not treated as a superuser. MapR Event Store For Apache Kafka does not allow the mapr user to run this command unless that user is given the relevant permission or permissions with access-control expressions. Syntax CAPTION: CLI maprcli stream purge -path REST http[s]://:/rest/stream/purge?path= Parameters CAPTION: Parameter Description path The path and name of the stream to reclaim disk space from.
