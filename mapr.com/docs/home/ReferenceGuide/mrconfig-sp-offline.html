mrconfig sp offline The mrconfig sp offline command takes a loaded storage pool offline. When a storage pool is offline it remains loaded into memory but it is not available to MapR-FS for reads and writes. The main use of the mrconfig sp offline command is to take a storage pool offline so the [1]fsck (filesystem check) command can be run on one or more disks or storage pools if there are lost or corrupt containers, directories, tables, files, filelets, or blocks. After running [2]fsck the storage pool is brought back online with the [3]mrconfig sp online command, and then typically the [4]gfsck (global filesystem check) command would be run on the affected cluster, volumes, or snapshots. See [5]mrconfig for instructions on running mrconfig commands. Syntax mrconfig sp offline Parameters Parameter Description sp path The device path of the storage pool; example /dev/sdc Examples Offline a loaded storage pool with a path of /dev/sdc on localhost /opt/mapr/server/mrconfig sp offline /dev/sdc References 1. file://localhost/root/docsync/tmp/mapr.com/docs/home/ReferenceGuide/fsck.html 2. file://localhost/root/docsync/tmp/mapr.com/docs/home/ReferenceGuide/fsck.html 3. file://localhost/root/docsync/tmp/mapr.com/docs/home/ReferenceGuide/mrconfig-sp-online.html 4. file://localhost/root/docsync/tmp/mapr.com/docs/home/ReferenceGuide/gfsck.html 5. file://localhost/root/docsync/tmp/mapr.com/docs/home/ReferenceGuide/mrconfig.html
