Configuring for Static and Dynamic Provisioning Using MapR Container Storage Interface (CSI) Storage Plugin This page summarizes the high-level steps for configuring the MapR Container Storage Interface (CSI) Storage Plugin after [1]installation to provide static or dynamic provisioning. To learn more about static and dynamic provisioning, see [2]Static and Dynamic Volume Provisioning Using MapR Container Storage Interface (CSI) Storage Plugin. Static Provisioning 1. Create the ticket secret and deploy the secret in the Pod only if the volume is on a secure MapR cluster. See [3]Configuring a Secret for more information. 2. Configure a PersistentVolume in your Pod spec or as part of a separate configuration file and provide information about the MapR volume. See [4]Example: Statically Provisioning a MapR Volume Using MapR Container Storage Interface (CSI) Storage Plugin, [5]Persistent Volumes, and [6]Example: Mounting a PersistentVolume for Static Provisioning Using MapR Container Storage Interface (CSI) Storage Plugin. 3. Configure a PersistentVolumeClaim in your Pod spec or as part of a separate configuration file. See [7]PersistentVolumeClaims. 4. Run the Pod spec using kubectl commands. See [8]Overview of kubectl. Dynamic Provisioning 1. Create the REST and ticket secrets and deploy the secrets in the Pod only if the volume is on a secure MapR cluster. See [9]Configuring a Secret for more information. 2. Create a storage class in your Pod spec or in a separate configuration file. See [10]Storage Classes and [11]Example: Mounting a PersistentVolume for Dynamic Provisioning Using MapR Container Storage Interface (CSI) Storage Plugin. 3. Configure a PersistentVolumeClaim in your Pod spec or in a separate configuration file. See [12]Example: Mounting a PersistentVolume for Dynamic Provisioning Using MapR Container Storage Interface (CSI) Storage Plugin. 4. Run the Pod spec using kubectl commands. See [13]Overview of kubectl.
