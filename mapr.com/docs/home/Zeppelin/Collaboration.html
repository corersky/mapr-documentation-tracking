Sharing Zeppelin Notebook Content By default, Zeppelin stores notebooks in the local file system in your container. An alternative is to store them in MapR-FS. This allows you to share the notebooks with other users. To store notebooks in MapR-FS, include the following environment variables in your [1]docker run command: Environment Variable Details MAPR_MOUNT_PATH The MapR-FS mount point. See [2]FUSE-Based POSIX Client for details about other related parameters you must provide. ZEPPELIN_NOTEBOOK_DIR The directory for storing Zeppelin notebooks. Specify a MapR-FS directory path. See [3]Notebook Storage for further details. See [4]https://zeppelin.apache.org/docs/latest/storage/storage.html for information about other available storage options.
